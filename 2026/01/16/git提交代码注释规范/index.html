常用提交类型说明

1️⃣ feat（新增功能）

用于 新增功能或需求。

2️⃣ fix（修复 bug）

用于 修复问题或缺陷。

3️⃣ perf（性能优化）

用于 性能或体验相关优化，不涉及功能变更。

4️⃣ refactor（代码重构）

用于 不影响功能、不修复 bug 的代码结构调整。

5️⃣ test（测试相关）

用于 新增或修改测试用例，包括单元测试、集成测试等。

6️⃣ docs（文档更新）

仅用于 文档类修改，不涉及代码逻辑。

7️⃣ style（代码风格调整）

仅修改 格式、空格、缩进、变量命名等，不改变逻辑。

8️⃣ chore（杂项 / 工程配置）

用于 构建流程、依赖、工具配置等变更。

9️⃣ revert（回滚）

用于 回滚到之前的提交，一般由 Git 自动生成。

🔟 merge（代码合并）

用于 分支合并或同步代码。
