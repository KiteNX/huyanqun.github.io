macOS 安装与使用 nvm（Node Version Manager）

macOS 安装与使用 nvm（Node.js 版本管理工具）

在 macOS 上进行前端或 Node.js 开发时，经常会遇到这样的问题：

  • 不同项目需要不同的 Node.js 版本
  • 升级 Node 后，老项目无法运行
  • 全局安装 Node，版本切换非常麻烦

👉 这时候就需要 nvm（Node Version Manager）。

nvm 可以让你在一台 Mac 上 同时安装多个 Node.js 版本，并随时切换

安装前准备

1️⃣ 确认系统环境

  • arm64 → Apple Silicon（M1 / M2 / M3）
  • x86_64 → Intel 芯片

nvm 同时支持 Intel 和 Apple 芯片，无需特殊处理

2️⃣ 确保已安装 Xcode Command Line Tools

如果已安装，会提示已存在；否则按提示安装即可。

1️⃣ zsh 用户（macOS 默认）

如果出现报错 source：no such file or directory: /User/xxx/.zshrc

  • .zshrc 是 zsh 的配置文件
  • 部分 Mac 系统（尤其是新系统或刚切换 zsh 的环境）默认不会自动创建 .zshrc

常用 nvm 命令汇总

命令 说明
nvm -v 查看 nvm 版本
nvm ls 查看本地 Node 版本
nvm ls-remote 查看远程 Node 版本
nvm install <version> 安装 Node
nvm use <version> 切换 Node
nvm alias default <version> 设置默认版本
nvm uninstall <version> 卸载 Node

总结

  • nvm 是 macOS 上 最推荐的 Node.js 管理方式
  • 支持多版本并存、快速切换
  • 不污染系统环境
  • 非常适合前端 / Node / 全栈开发

👉 结论：在 Mac 上，永远用 nvm 管理 Node.js
